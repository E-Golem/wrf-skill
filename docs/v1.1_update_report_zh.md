# v1.1.1 更新报告

## 版本

- 版本号：`1.1.1`
- Git tag：`v1.1.1`

## 本次修订

- README、示例文档和发布说明改为面向外部用户的通用写法。
- 示例工作目录统一使用 `D:/wrf/wrf-station-verification`。
- 推荐 conda 环境名统一为 `wrfpy`。
- 默认边界文件示例改为 `data/boundary/study_area.shp`。
- 移除主分支中已跟踪的历史本地验证报告，避免公开仓库包含个人或项目内部验证结果。
- 更新 `.gitignore`，默认忽略 `data/`、`outputs/`、`reports/`、`dist/` 和构建产物。
- 清理旧版单方案脚本，新增通用批处理脚本 `scripts/run_wrf_score.ps1`。
- 新增公共文档测试，防止本机路径、旧环境名和已删除 CLI 参数再次进入发布文件。

## v1.1.x 功能范围

- 输入类型固定为 WRF `wrfout`。
- 时间规则固定为 WRF UTC 转北京时 UTC+8，日界固定为北京时 20:00。
- 输出目录统一为 `outputs/runs/<run_id>/`。
- 固定评价指标为 `PCC, bias, MAE, RMSE, Normalized cRMSE, RSD`。
- 支持 `Tmax`、`T2`、`RH` 三类验证变量。
- 支持完整日识别和 `excluded_days.csv`。

## 数据要求

- `Tmax/T2` 验证需要 `SURF_CLI_CHN_MUL_DAY-TEM-*.TXT`。
- `RH` 验证需要 `SURF_CLI_CHN_MUL_DAY-RHU-*.TXT`。
- `RH` wrfout 输入需要 `RH2`、`RH`，或同时包含 `T2, Q2, PSFC`。
