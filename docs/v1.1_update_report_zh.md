# v1.1 更新报告

## 版本

- 版本号：`1.1.0`
- Git tag：`v1.1`

## 主要变化

- 删除旧的 `input-kind`、`report-dir`、`score-metrics`、时间偏移和日界调用参数。
- 输入统一为 WRF `wrfout` 结构。
- 时间规则固定为 WRF UTC 转北京时 UTC+8，日界固定为北京时 20:00。
- 输出目录合并为 `outputs/runs/<run_id>/`，报告和 CSV 不再分散到两个根目录。
- 固定评价指标为 `PCC, bias, MAE, RMSE, Normalized cRMSE, RSD`。
- 新增 `T2` 和 `RH` 验证能力，保留 `Tmax` 验证。
- 新增完整日识别和 `excluded_days.csv`。
- 新增 RHU 观测解析器。
- CLI 仅保留 `run`、`inspect`、`compare` 三个子命令。

## 数据要求

- `Tmax/T2` 验证需要 `SURF_CLI_CHN_MUL_DAY-TEM-*.TXT`。
- `RH` 验证需要 `SURF_CLI_CHN_MUL_DAY-RHU-*.TXT`。
- `RH` wrfout 输入需要 `RH2`、`RH`，或同时包含 `T2, Q2, PSFC`。

## 输出变化

所有输出进入：

```text
outputs/runs/<run_id>/
```

单方案输出包括：

```text
tables/overall_score.csv
tables/daily_scores.csv
tables/station_scores.csv
tables/excluded_days.csv
reports/wrf_<variable>_evaluation_report.md
```

批量比较输出包括：

```text
scheme-comparison/scheme_comparison.csv
scheme-comparison/scheme_comparison_report.md
```

## 注意事项

当前本地示例数据不包含 RHU 观测文件，部分示例 wrfout 也不包含 RH/Q2/PSFC，因此 RH 功能已经实现，但实际运行需要后续补充完整数据。
