# v1.1 Update Report

## Version

- Package version: `1.1.0`
- Git tag: `v1.1`

## Major Changes

- Removed the legacy public fields `input-kind`, `report-dir`, `score-metrics`, time offset, and day-boundary options.
- Standardized input to WRF `wrfout` files.
- Fixed time handling to WRF UTC -> Beijing time UTC+8 and a Beijing 20:00 station-day boundary.
- Merged outputs and reports under `outputs/runs/<run_id>/`.
- Fixed metrics to `PCC, bias, MAE, RMSE, Normalized cRMSE, RSD`.
- Added validation support for `T2` and `RH` while keeping `Tmax`.
- Added complete-day detection and `excluded_days.csv`.
- Added a China daily RHU observation parser.
- Simplified the CLI to `run`, `inspect`, and `compare`.

## Data Requirements

- `Tmax/T2` validation requires `SURF_CLI_CHN_MUL_DAY-TEM-*.TXT`.
- `RH` validation requires `SURF_CLI_CHN_MUL_DAY-RHU-*.TXT`.
- `RH` wrfout input requires `RH2`, `RH`, or the combination `T2, Q2, PSFC`.

## Output Changes

All outputs are written to:

```text
outputs/runs/<run_id>/
```

Per-scheme outputs include:

```text
tables/overall_score.csv
tables/daily_scores.csv
tables/station_scores.csv
tables/excluded_days.csv
reports/wrf_<variable>_evaluation_report.md
```

Batch comparison outputs include:

```text
scheme-comparison/scheme_comparison.csv
scheme-comparison/scheme_comparison_report.md
```

## Notes

The current local sample data does not include RHU observations, and some sample wrfout-derived files do not include RH/Q2/PSFC. RH validation is implemented but requires complete input data.
