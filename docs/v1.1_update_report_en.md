# v1.1.1 Update Report

## Version

- Package version: `1.1.1`
- Git tag: `v1.1.1`

## Changes in This Patch

- Rewrote README, examples, and release notes for public reuse.
- Standardized example workspace paths to `D:/wrf/wrf-station-verification`.
- Standardized the recommended conda environment name to `wrfpy`.
- Changed the example boundary path to `data/boundary/study_area.shp`.
- Removed tracked historical local verification reports from the main branch.
- Updated `.gitignore` to ignore `data/`, `outputs/`, `reports/`, `dist/`, and build artifacts.
- Removed outdated single-scheme scripts and added the portable batch script `scripts/run_wrf_score.ps1`.
- Added a public-docs test to prevent local paths, old environment names, and removed CLI options from re-entering published files.

## v1.1.x Functional Scope

- Input type is fixed to WRF `wrfout`.
- Time handling is fixed to WRF UTC -> Beijing time UTC+8 and a Beijing 20:00 station-day boundary.
- Output root is unified as `outputs/runs/<run_id>/`.
- Metrics are fixed to `PCC, bias, MAE, RMSE, Normalized cRMSE, RSD`.
- Supported validation variables are `Tmax`, `T2`, and `RH`.
- Complete-day detection and `excluded_days.csv` are supported.

## Data Requirements

- `Tmax/T2` validation requires `SURF_CLI_CHN_MUL_DAY-TEM-*.TXT`.
- `RH` validation requires `SURF_CLI_CHN_MUL_DAY-RHU-*.TXT`.
- `RH` wrfout input requires `RH2`, `RH`, or the combination `T2, Q2, PSFC`.

## Download Package

The v1.1.1 release package includes a Python wheel, a source zip, and a source tar.gz under `release-assets/v1.1.1/`. If a GitHub Release page is not available, users can also download the Source code archive from tag `v1.1.1`.
