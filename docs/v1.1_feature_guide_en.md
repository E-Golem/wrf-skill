# WRF Station Verification Toolkit v1.1 Feature Guide

## Purpose

v1.1 is an engineering-oriented WRF post-processing and station verification toolkit. It focuses on `wrfout` inputs, Chinese daily station observations, batch scoring, and automatic parameterization scheme ranking.

## Fixed Rules

- Input type: WRF `wrfout` files only.
- Observation type: Chinese meteorological daily station data.
- Time handling: WRF UTC timestamps are shifted to Beijing time UTC+8.
- Station-day boundary: Beijing time 20:00.
- Metrics: `PCC, bias, MAE, RMSE, Normalized cRMSE, RSD`.
- Output root: all tables and reports are written under `outputs/runs/<run_id>/`.

## Supported Variables

- `Tmax`: daily maximum 2 m temperature aggregated from hourly `T2`, compared with station `tmax_obs_c`.
- `T2`: daily mean 2 m temperature aggregated from hourly `T2`, compared with station `tmean_obs_c`.
- `RH`: daily mean 2 m relative humidity read from `RH2` or `RH`, or calculated from `T2 + Q2 + PSFC`, compared with RHU station observations.

## Complete-Day Detection

The tool infers the expected frame count per day from the input time step. Only complete station-days inside the validation window are scored. Excluded dates are written to:

```text
outputs/runs/<run_id>/<scheme>/tables/excluded_days.csv
```

The file includes `date, reason, actual_records, expected_records`.

## Recommended Use

- Batch comparison of longwave and shortwave radiation parameterization experiments.
- Adding an applied engineering workflow to WRF-based thesis work.
- Building a foundation for later visualization, dashboards, and automated reporting.

## Current Limitations

- Only Chinese station daily observation files are supported.
- The current local sample data does not include RHU observations or full RH/Q2/PSFC wrfout variables.
- Station sampling uses nearest WRF grid cell; bilinear interpolation and elevation correction are future extensions.
